# Personal Meeting Notes

TypeScript-first monorepo for a privacy-focused desktop assistant that detects meetings, captures notes (with consent), summarizes content, and persists action items.

## Architecture

```mermaid
flowchart LR
  subgraph DesktopApp[apps/desktop (Tauri + React)]
    UI[React UI]
    DetectorController[Detection Controller]
    Consent[Consent + Session Local Storage]
  end

  subgraph Core[packages/core]
    DetectionSvc[BackgroundMeetingDetectionService]
    Pipeline[MeetingProcessingPipeline]
    Repo[PersistenceRepository]
    DB[(SQLite via better-sqlite3)]
    Secrets[OS Secret Store]
    Config[Env Config Validation]
  end

  subgraph AIProviders[External AI Providers]
    Voxtral[Voxtral Mini Transcribe V2]
    Mistral[Mistral Small 4]
  end

  UI --> DetectorController
  DetectorController --> DetectionSvc
  DetectorController --> Consent
  DetectionSvc --> Pipeline
  Pipeline --> Voxtral
  Pipeline --> Mistral
  Pipeline --> Repo
  Repo --> DB
  Pipeline --> Secrets
  Pipeline --> Config
```

### Key packages

- `apps/desktop`: Tauri + React shell with meeting detection flow and user consent handling.
- `packages/core`: core detection services, processing pipeline, storage, config, and security primitives.

## Local setup and run

### Prerequisites

- Node.js 20+
- npm 10+
- Rust toolchain (required for `tauri:dev` / `tauri:build`)

### Install

```bash
npm install
```

### Configure environment

1. Copy the template:

```bash
cp .env.example .env
```

2. Fill in non-secret config values in `.env`.
3. Store API credentials/tokens in your OS keychain via the app secret-store integration (do **not** commit secrets).

### Run in development

- Frontend-only dev loop:

```bash
npm run dev
```

- Full desktop app (Tauri):

```bash
npm run tauri:dev --workspace @meetings/desktop
```

### Build

- Frontend build:

```bash
npm run build
```

- Desktop package build:

```bash
npm run tauri:build --workspace @meetings/desktop
```

## Permissions and privacy model

### Detection permissions

- Meeting detection runs through adapter snapshots (Teams/Zoom) and emits only state transitions (`meeting_started`, `meeting_ended`, `meeting_unknown`).
- Detection does not require transcript capture by default.

### Capture permissions

- Capture consent is tracked per-platform in browser local storage (`meeting-consent:<platform>`).
- Session state for in-progress captures is stored separately and is cleared on end/reset.

### Data handling and storage

- Structured meeting artifacts are persisted locally in SQLite (`meetings`, `transcripts`, `summaries`, `notes`, `action_items`).
- Failed processing stages are saved to `dead_letters` for debugging/retry.
- Secrets are expected to live in OS-native credential storage, not source control or plaintext config.

### Privacy defaults

- No secret placeholders are committed with values.
- `.env.example` includes only non-secret placeholders.
- API tokens should be injected securely at runtime.

## Deployment and packaging

### CI-ready checks

Use these checks before packaging:

```bash
npm run lint
npm run typecheck
npm run test
```

### Desktop packaging targets

- macOS: `.app` / `.dmg` via Tauri bundling
- Windows: installer executable via Tauri bundling
- Linux: distro-compatible package artifacts

Artifacts are generated by:

```bash
npm run tauri:build --workspace @meetings/desktop
```

### Release flow (recommended)

1. Run lint/typecheck/test in CI.
2. Build Tauri artifacts on platform-specific runners.
3. Sign binaries per platform requirements.
4. Publish checksums + release notes.
